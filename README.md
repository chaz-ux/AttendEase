## рҹ“Ў &nbsp;What Is This?

Imagine a lecturer walks into a room, opens an app, taps one button вҖ” and attendance is **open**. The app captures their GPS position and draws an invisible fence around the room. Every student who walks in, pulls out their phone, and marks present gets verified by the server in real time. If they're outside the fence? Rejected. Different device? Rejected. Already checked in? Rejected.

That's AttendEase. Built for **JKUAT**, designed for scale.

## рҹ—„ &nbsp;Database Schema

```sql
universities в”Җв”Җв”¬в”Җв”Җ buildings в”Җв”Җ classrooms
               в”Ӯ
               в”ңв”Җв”Җ profiles (students / lecturers / admins)
               в”Ӯ       в”Ӯ
               в”Ӯ       в”ңв”Җв”Җ units (lecturer_id вҶ’ profiles)
               в”Ӯ       в”Ӯ       в”Ӯ
               в”Ӯ       в”Ӯ       в””в”Җв”Җ sessions (unit_id вҶ’ units)
               в”Ӯ       в”Ӯ               в”Ӯ
               в”Ӯ       в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв””в”Җв”Җ attendance_logs
               в”Ӯ                           (student_id + session_id)
               в””в”Җв”Җ attendance_logs (university_id)
```

### Key Design Decisions

| Decision | Reason |
|----------|--------|
| `center_lat/lng` on `sessions` | Lecturer's live GPS = geofence center. No pre-configured rooms needed. |
| `room_label` (free text) | Venues change. Flexibility beats rigidity. |
| `device_id` on `attendance_logs` | Anti-proxy: one device per student account |
| `unique(session_id, student_id)` | Database-enforced no double check-ins |
| Edge function verification | Client GPS can be spoofed. Server re-checks. |

## рҹ”җ &nbsp;Security Model

```
THREAT: Student asks friend to check in for them
DEFENCE: Device fingerprinting вҖ” account locked to registered device

THREAT: Student fakes GPS location
DEFENCE: Server-side re-verification in Edge Function

THREAT: Double check-in
DEFENCE: Unique constraint on (session_id, student_id)

THREAT: Unauthorized data access
DEFENCE: Row Level Security вҖ” every query filtered by auth.uid()

THREAT: Stolen credentials
DEFENCE: Biometric second factor вҖ” must physically hold registered device
```
